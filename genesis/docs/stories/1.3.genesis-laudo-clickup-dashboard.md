## Status: Done

## Story

**As a** usuário do genesis-laudo,
**I want** ver no dashboard a fila real de cards ClickUp ("para fazer") e as análises em andamento ("em processo"),
**so that** eu possa monitorar e iniciar análises diretamente pela UI.

## Acceptance Criteria

1. `app/services/clickup.js` encapsula chamadas à ClickUp REST API (token via env var)
2. `GET /laudo/` busca cards reais da lista `901324787605` com status "para fazer"
3. Dashboard exibe cards da fila com: nome da marca, cliente, atividade
4. Dashboard exibe análises ativas ("em processo") com fase atual (lida do PostgreSQL)
5. Dashboard exibe análises concluídas ("feito") com links para documentos
6. Botão "Iniciar análise" no card da fila chama `POST /laudo/analise/nova` (stub — implementado na Story 1.4)
7. Dashboard atualiza sem reload completo via polling simples (5 segundos) ou botão refresh
8. Erro de ClickUp API → dashboard mostra fila vazia com mensagem de aviso (não quebra)

## Dev Notes

ClickUp API v2: `GET /api/v2/list/901324787605/task?statuses[]=para fazer`
Autenticação: `Authorization: {CLICKUP_API_TOKEN}` (header, não Bearer)
Padrão curl do projeto (genesis-protocolo): temp JSON file + curl -d @file para evitar escaping.
No Node.js usar `fetch` nativo (Node 20+) com headers.

Analyses "ativas" vêm do PostgreSQL (fase 1-3), não do ClickUp.
Analyses "concluídas" combinam PostgreSQL (fase=4) + links dos arquivos.
