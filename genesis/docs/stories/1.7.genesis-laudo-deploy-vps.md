## Story

**As a** time da Vialum,
**I want** o genesis-laudo rodando na VPS vps-nova com TLS e URL pública,
**so that** qualquer membro da equipe possa acessar o dashboard pelo browser.

## Acceptance Criteria

1. Imagem Docker construída na VPS sem erros
2. `genesis-laudo-db` PostgreSQL sobe com volume persistente (`genesis-laudo-db-data`)
3. Migration `001_init.sql` executada automaticamente no primeiro boot
4. `genesis-laudo` app sobe, conecta ao DB, ouve na porta 3001
5. Traefik roteia `api.luminai.ia.br/laudo/*` para o container (PathPrefix)
6. TLS via Let's Encrypt funcional (HTTPS)
7. `curl https://api.luminai.ia.br/laudo/health` retorna `{"status":"ok"}`
8. Dashboard acessível no browser em `https://api.luminai.ia.br/laudo/`
9. Credenciais Google Drive copiadas para `/root/genesis-laudo/app/resources/`
10. `.env` criado na VPS com DB_PASSWORD e CLICKUP_API_TOKEN

## Dev Notes (expandidas)

- VPS usa `docker-compose` v1.29.2 (não `docker compose` v2)
- Container roda como usuário `node` (uid 1000) — obrigatório pois `--dangerously-skip-permissions` é bloqueado como root
- Volume `.claude`: montado em `/home/node/.claude` (HOME do usuário node no container)
- Volume `genesis`: `/root/genesis:/root/genesis` (aiOS workspace + scripts)
- Volume `laudos`: `/root/laudos:/root/laudos` com `chmod 777` no host para escrita pelo uid 1000
- Labels Traefik: precisa de `traefik.docker.network=proxy` (container em 2 redes) e `Host()` na regra
- Resolver TLS: `le` (não `letsencrypt`)
- `WORKSPACE_PATH=/root` → scripts em `/root/genesis/squads/...`, laudos em `/root/laudos/...`
