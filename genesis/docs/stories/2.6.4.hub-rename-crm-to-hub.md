# Story 2.6.4: Rename CRM Hub → Vialum Hub

## Status: Draft

## Executor
executor: "@dev"
quality_gate: "@qa"

## Story
**As a** desenvolvedor do ecossistema Vialum,
**I want** que o CRM Hub seja renomeado para Vialum Hub em todo o ecossistema,
**so that** o nome reflita sua função real (hub de identidade universal, não só CRM).

## Acceptance Criteria
1. [ ] Models Prisma renomeados: CrmContact → HubContact, CrmIntegration → HubIntegration
2. [ ] Tabelas mantêm nomes antigos via @@map (zero migration de dados)
3. [ ] Rotas: /crm/* e /hub/* funcionam em paralelo (dual route period)
4. [ ] Container Docker: vialum-crm-hub → vialum-hub
5. [ ] Traefik route: PathPrefix(/hub) adicionado (manter /crm como alias)
6. [ ] ENV vars: CRM_HUB_URL → HUB_URL em todos os consumidores (Chat, Switch, etc.)
7. [ ] Manter backward compatibility por 30 dias (rotas /crm continuam funcionando)
8. [ ] Documentação atualizada (PRDs, stories, README)

## Tasks
- [ ] Hub Prisma: renomear models com @@map mantendo tabelas
- [ ] Hub src: rename imports e referências CrmContact → HubContact
- [ ] Hub routes: adicionar prefixo /hub como alias de /crm
- [ ] Hub docker-compose: renomear container e service
- [ ] Chat env: CRM_HUB_URL → HUB_URL (manter CRM_HUB_URL como fallback)
- [ ] Atualizar todos os docker-compose.yml com novo container name
- [ ] Atualizar documentação e PRDs

## Dev Notes
- Fase cosmética. Não alterar funcionalidade nenhuma.
- @@map do Prisma permite renomear models sem renomear tabelas no banco.
- O dual route period (30 dias) evita quebrar serviços que ainda usam /crm.
- Fazer DEPOIS de todas as mudanças funcionais estarem estáveis.

## Dependências
- Stories 2.6.0, 2.6.1, 2.6.2, 2.6.3 todas concluídas e estáveis.
