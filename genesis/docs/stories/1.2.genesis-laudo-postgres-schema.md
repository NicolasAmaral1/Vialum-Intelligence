## Story

**As a** desenvolvedor do projeto Vialum,
**I want** um schema PostgreSQL idempotente com 3 tabelas de rastreamento,
**so that** toda operação do genesis-laudo seja auditável e o sistema seja escalável.

## Acceptance Criteria

1. Migration `001_init.sql` em `genesis-laudo/migrations/` com todas as DDLs usando `IF NOT EXISTS`
2. Tabela `analyses`: rastreia análises (1:1 com task_id ClickUp, UNIQUE garante idempotência)
3. Tabela `analysis_events`: log imutável append-only de eventos por fase
4. Tabela `cli_executions`: rastreia cada chamada ao claude CLI (command, status, output, timing)
5. Trigger `set_updated_at()` atualiza `updated_at` automaticamente em `analyses`
6. Coluna `idempotency_key VARCHAR UNIQUE` reservada para webhooks futuros
7. `app/services/db.js` exporta pool pg + helpers: `analyses.*`, `events.insert`, `cli.insert/finish`
8. `waitForDb()` aguarda Postgres (max 10 tentativas com backoff) antes de ouvir na porta
9. Sem `DATABASE_URL` → servidor sobe normalmente com ⚠️ log (modo dev)
10. Migration montada em `/docker-entrypoint-initdb.d` — Postgres executa automaticamente no primeiro boot
