# Story 2.5.3: Vialum Chat — Provider Abstraction Cleanup

## Status: Done

## Executor
executor: "@dev"
quality_gate: "@qa"

## Story
**As a** desenvolvedor do Vialum Chat,
**I want** que toda lógica específica de provider (Evolution API, Cloud API) esteja
encapsulada nos adapters,
**so that** adicionar um novo provider de WhatsApp exija apenas criar 1 arquivo adapter
sem modificar workers, webhooks ou serviços.

## Acceptance Criteria
1. [x] `IWhatsAppProvider` interface expandida com `normalizeIncomingWebhook()`, `fetchProfilePicture()`, `sendPresence()`
2. [x] `EvolutionAdapter` implementa: normalização de payload, resolução de LID, extração de conteúdo, fetch de avatar, presença
3. [x] `CloudApiAdapter` implementa: normalização de payload Cloud API, extração de conteúdo
4. [x] `webhook-process.worker.ts` usa `provider.normalizeIncomingWebhook()` — zero código provider-specific
5. [x] `message-send.worker.ts` usa `provider.sendText()` — removidas `sendViaEvolutionAPI()` e `sendViaCloudAPI()`
6. [x] Typing presence usa `provider.sendPresence()` — removido hardcode `if (provider === 'evolution_api')`
7. [x] `fetchAndUpdateAvatar` usa `provider.fetchProfilePicture()` — provider-agnostic
8. [x] Dedup de mensagem atômico via `$transaction`
9. [x] Fire-and-forget tasks logam erros com `console.error` em vez de `.catch(() => {})`
10. [x] Zero ocorrências de `if (provider === 'evolution_api')` fora dos adapters
