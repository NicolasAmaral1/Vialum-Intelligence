# Story 2.6.3: Vialum Hub — Proactive Sync + External ID Aliases

## Status: Draft

## Executor
executor: "@dev"
quality_gate: "@qa"

## Story
**As a** operador do Vialum,
**I want** que o Hub importe contatos dos providers periodicamente e salve IDs externos como aliases,
**so that** um contato no ClickUp seja automaticamente vinculado ao mesmo contato no Pipedrive
e no WhatsApp, sem intervenção manual.

## Acceptance Criteria
1. [ ] Job periódico (cron) que importa contatos/deals ativos de cada provider por tenant
2. [ ] Stagger per-tenant: não sincroniza todos os tenants ao mesmo tempo
3. [ ] Rate limit per-provider per-tenant respeitado (usa capabilities do provider)
4. [ ] Após sync, IDs externos salvos como aliases:
   - `pipedrive_person:12345`
   - `pipedrive_deal:67890`
   - `clickup_member:abc`
   - `rdstation_contact:def`
5. [ ] Resolução reversa: `POST /identity/resolve { externalId: "12345", provider: "pipedrive" }` encontra o contato
6. [ ] ClickUp members importados com email/nome → linkados a HubContacts existentes por email
7. [ ] Dedup: se contato já existe (por phone ou email), não cria duplicata — linka
8. [ ] Dashboard: admin vê último sync por provider por tenant + contagem de contatos

## Tasks
- [ ] Hub: criar worker cron `proactive-sync.worker.ts` (ex: a cada 2h)
- [ ] Hub: per-tenant stagger (distribuir tenants no intervalo do cron)
- [ ] Hub: após sync, salvar external IDs como ContactAlias
- [ ] Hub: dedup na importação — buscar por email/phone antes de criar
- [ ] Hub: ClickUp member import → buscar por email no alias table
- [ ] Hub: endpoint GET /admin/sync-status por tenant
- [ ] Hub: identity.service resolve por externalId + provider (já parcialmente existe)

## Dev Notes
- O proactive sync NÃO substitui o on-demand sync (que roda quando contato é acessado).
- Ele complementa: garante que contatos que nunca foram acessados no Chat também existam no Hub.
- O stagger distribui tenants uniformemente: se cron roda a cada 2h e tem 100 tenants, cada tenant sincroniza a cada 2h mas em momentos diferentes (tenant 1 no minuto 0, tenant 2 no minuto 1.2, etc.)
- ClickUp não tem busca por phone — import é por listagem de members + match por email/nome.

## Dependências
- Story 2.6.1 (Hub Identity Foundation) — precisa de alias extensível e sync via queue.
