# Story 2.6.2: Vialum Chat — Context API + Active Channels

## Status: Draft

## Executor
executor: "@dev"
quality_gate: "@qa"

## Story
**As a** IA ou operador do Vialum Chat,
**I want** uma API que retorne o contexto completo de uma conversa (mensagens + CRM + canal ativo)
e que eu consiga marcar qual é o canal principal de um contato por inbox,
**so that** a IA saiba onde responder e tenha todo o contexto pra tomar decisões.

## Acceptance Criteria
1. [ ] `ContactInbox` ganha campos: `activeConversationId` (nullable), `linkedGroupId` (nullable)
2. [ ] Canal ativo e grupo vinculado são **per-inbox** (não per-contact global) — mesmo contato pode ter canais diferentes em inboxes diferentes
3. [ ] Endpoint `GET /conversations/:id/context` retorna:
   - Dados da conversa (status, inbox, labels, assignee)
   - Contato com displayName, formattedPhone, hubContactId
   - Últimas N mensagens (parâmetro `limit`, default 30)
   - Opção `includeTextContent` (default false) — pra IA que precisa ler mídia
   - Grupo vinculado (se existir) com membros
   - Talk/TreeFlow ativo
   - CRM data via Hub — com timeout 500ms + cache local 5min
   - Campo `activeChannel: 'group' | 'individual' | null`
   - Se conversa 1:1 e contato tem grupo vinculado: resumo do grupo (últimas 5 msgs)
   - Se conversa grupo e contato tem 1:1: resumo do 1:1 (últimas 5 msgs)
   - Campo `hint` com contexto pra IA (ex: "Canal principal é o grupo. Mensagem chegou no privado.")
4. [ ] Endpoint `PATCH /contacts/:id/channel` pra marcar canal ativo e grupo vinculado (per-inbox)
5. [ ] Se Hub estiver fora do ar ou timeout, context retorna sem CRM data (graceful degradation)
6. [ ] Cache local de CRM data no Chat (Redis, 5min TTL) — não chama Hub toda request
7. [ ] WhatsAppGroupMapping migration rodada no Hub
8. [ ] Quando operador vincula grupo a contato, cria WhatsAppGroupMapping no Hub via HTTP

## Dev Notes
- Canal ativo e grupo vinculado são no `ContactInbox`, NÃO no `Contact`. Motivo: mesmo contato pode ter inbox do Luan (canal = grupo) e inbox do Nicolas (canal = 1:1).
- O cache local de CRM evita chamadas repetidas ao Hub. Key: `crm-context:{phone}:{accountId}`. TTL 5min.
- O timeout de 500ms é curto o suficiente pra não atrasar a IA, longo o suficiente pra Hub responder na maioria dos casos.
- O `hint` é texto livre gerado pelo context.service.ts — a IA usa como contexto, não como regra.
- O dropdown de vincular grupo mostra SOMENTE grupos onde o contato é membro (JOIN GroupMember).
