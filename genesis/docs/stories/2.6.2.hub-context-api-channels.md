# Story 2.6.2: Vialum Chat — Context API + Active Channels

## Status: Draft

## Executor
executor: "@dev"
quality_gate: "@qa"

## Story
**As a** IA ou operador do Vialum Chat,
**I want** uma API que retorne o contexto completo de uma conversa (mensagens + CRM + canal ativo)
e que eu consiga marcar qual é o canal principal de um contato (grupo ou 1:1),
**so that** a IA saiba onde responder e tenha todo o contexto pra tomar decisões.

## Acceptance Criteria
1. [ ] Contact ganha campos: `activeConversationId` (nullable), `linkedGroupId` (nullable)
2. [ ] `activeConversationId` pode ser conversa 1:1 ou de grupo — indica o canal preferido
3. [ ] `linkedGroupId` aponta pro grupo "oficial" do contato (se existir)
4. [ ] Endpoint `GET /conversations/:id/context` retorna:
   - Dados da conversa (status, inbox, labels, assignee)
   - Contato com displayName e formattedPhone
   - Últimas N mensagens com textContent (default 30)
   - Grupo vinculado (se existir) com membros
   - Talk/TreeFlow ativo
   - CRM data via Hub (deals, tasks, docs) — com timeout 2s + fallback
   - Campo `activeChannel` indicando se este é o canal principal ou não
   - Se conversa 1:1 e contato tem grupo vinculado: resumo do grupo
   - Se conversa grupo e contato tem 1:1: resumo do 1:1
5. [ ] Endpoint `PATCH /contacts/:id/channel` pra marcar canal ativo e grupo vinculado
6. [ ] Se Hub estiver fora do ar, context retorna sem CRM data (graceful degradation)
7. [ ] WhatsAppGroupMapping migration rodada no Hub
8. [ ] Quando operador vincula grupo a contato, cria WhatsAppGroupMapping no Hub

## Dev Notes
- O context endpoint vive no Chat, NÃO no Hub. Chat é o dono de conversas e mensagens.
- A chamada ao Hub é com timeout 2s. Se falhar, retorna context parcial (sem CRM).
- O `activeConversationId` é setado manualmente pelo operador. A IA consulta mas não muda.
- O `linkedGroupId` é setado quando operador vincula grupo ao contato.
- A IA recebe `hint: "Canal principal é o grupo. Esta mensagem chegou no privado."` quando apropriado.
- WhatsAppGroupMapping no Hub linka groupJid → HubContact (pra identity resolution por grupo).
