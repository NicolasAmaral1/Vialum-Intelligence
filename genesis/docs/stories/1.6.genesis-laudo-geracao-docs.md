# Story 1.6: Geração PDF/DOCX + SSE + Drive + ClickUp finalização

## Status: Done

## Executor
executor: "@dev"

## Story

**As a** analista da Vialum,
**I want** gerar os documentos finais (PDF + DOCX) pela UI e ver o progresso em tempo real,
**so that** o laudo seja entregue ao cliente sem precisar usar o terminal.

## Acceptance Criteria

1. `POST /laudo/gerar` executa `@laudo *gerar` via Claude CLI
2. SSE endpoint `GET /laudo/analise/:task_id/stream` emite eventos de progresso durante geração
3. UI exibe barra de progresso com etapas: "Gerando PDF... Gerando DOCX... Enviando ao Drive... Atualizando ClickUp..."
4. Após conclusão: links para PDF, DOCX e Google Drive exibidos na página de análise
5. `analyses.output_pdf`, `analyses.output_docx`, `analyses.drive_url` atualizados no PostgreSQL
6. Card ClickUp movido para "feito" automaticamente
7. `analyses.fase` atualizado para 4 (concluída)
8. Falha em PDF → continua com DOCX, reporta erro mas não bloqueia
9. Falha no Drive → reporta, mantém arquivos locais acessíveis

## Dev Notes

SSE: `res.writeHead(200, {'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache'})`.
Eventos emitidos: `data: {"step":"pdf","status":"running"}\n\n`

Claude CLI output parseado para extrair paths dos arquivos gerados.
Alternativa: após CLI concluir, listar arquivos em `laudos/{cliente}/{marca}/` e detectar PDF/DOCX.
