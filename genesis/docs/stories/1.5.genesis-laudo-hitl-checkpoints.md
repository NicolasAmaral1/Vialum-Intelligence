## Story

**As a** analista da Vialum,
**I want** visualizar e aprovar os 2 checkpoints da análise de viabilidade na UI,
**so that** eu possa revisar o trabalho da Mira antes de gerar os documentos finais.

## Acceptance Criteria

1. `GET /laudo/analise/:task_id` exibe stepper visual com fase atual (0-4)
2. Na fase 2: PARTE 1 renderizada em HTML a partir do `{marca} - PLANO DE ANÁLISE.md`
3. `POST /laudo/checkpoint/1` aprova PARTE 1: `analyses.checkpoint_1='aprovado'`, fase → 3, executa `@laudo *inpi`
4. Na fase 3: textarea para paste dos dados INPI + botão "Processar"
5. `POST /laudo/inpi` recebe texto INPI → escreve em `inpi-raw.txt` → executa `processar_inpi.py`
6. Após processamento INPI: PARTE 2 renderizada para revisão
7. `POST /laudo/checkpoint/2` aprova PARTE 2: `analyses.checkpoint_2='aprovado'`, fase → 4
8. Checkpoints são bloqueantes: sem aprovação, sistema não avança (veto no backend)
9. Markdown renderizado via biblioteca `marked` (já no package.json)
10. SSE ou polling para mostrar "Aguardando Claude..." durante execução do CLI
