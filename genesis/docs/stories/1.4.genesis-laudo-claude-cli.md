## Story

**As a** sistema genesis-laudo,
**I want** executar o Claude Code CLI como subprocess e registrar cada execução no PostgreSQL,
**so that** o squad @laudo processe as análises no servidor enquanto o dashboard rastreia o progresso.

## Acceptance Criteria

1. `app/services/claude.js` executa `claude -p "@laudo *{comando}" --dangerously-skip-permissions` via `child_process.spawn`
2. Cada execução cria registro em `cli_executions` (status=running) antes de iniciar
3. Ao concluir (sucesso ou falha), `cli_executions` é atualizado (status, output, exit_code, finished_at)
4. `POST /laudo/analise/nova` cria registro em `analyses` (upsert idempotente por task_id) e executa `*coletar`
5. Claude escreve arquivos em `laudos/{cliente}/{marca}/` — path compartilhado via volume Docker
6. Após `*coletar` concluir: `analyses.fase` atualizado para 2, evento registrado em `analysis_events`
7. Timeout de 5 minutos por execução CLI — se exceder, status=timeout e usuário é notificado
8. WORKSPACE_PATH env var define o caminho do workspace do aiOS na VPS (`/root/genesis`)

## Dev Notes

CLI path na VPS: `claude` (global, via npm install -g @anthropic-ai/claude-code ou similar).
WORKSPACE_PATH=/root/genesis — o claude CLI roda neste diretório.
Comando exemplo: `claude -p "@laudo *coletar --task-id 86af3jfbm" --dangerously-skip-permissions`

O arquivo `laudos/{cliente}/{marca}/{marca} - PLANO DE ANÁLISE.md` é escrito pelo CLI.
O dashboard lê este arquivo para exibir nos checkpoints.
