## Dev Notes

### Auth service-to-service
- Chat autentica no Hub usando MEDIA_JWT_SECRET (que Г© o JWT_SECRET do Hub: `vialum-crm-hub-secret-2024`)
- Gera JWT: `{ userId: 'chat-service', accountId, role: 'service' }`, expira em 5min
- Mesmo padrГЈo que Switch usa pra chamar Media Service

### Redis compartilhado
- Hub usa o Redis do Chat (vialumchat-redis na rede vialumchat-internal)
- Filas BullMQ sГЈo namespaced por nome вҖ” sem colisГЈo
- Job data deve ser pequeno (sГі IDs, nГЈo rawData) pra nГЈo impactar Chat

### Race condition prevention
- `findOrCreateContact` usa advisory lock no PostgreSQL: `pg_advisory_xact_lock(hashtext(accountId || phone))`
- Ou alternativa: INSERT alias com ON CONFLICT, usar o resultado pra determinar se contato jГЎ existia

### Name priority
```
PRIORITY = { whatsapp: 1, chat: 2, task: 3, crm: 4, manual: 5 }
if (newPriority >= existingPriority) updateName()
```

### Alias normalization
- Pipedrive: externalId `person-123` вҶ’ alias type `pipedrive_person`, value `123`
- RD Station: externalId `contact-456` вҶ’ alias type `rdstation_contact`, value `456`
- Strip o prefixo `{resourceType}-` pra obter o raw ID

### Bug existente corrigido nesta story
- `syncContactNameFromCRM` envia request sem JWT вҶ’ sempre retorna 401 silenciosamente
- SubstituГӯdo pelo hub-ensure worker que usa JWT correto
